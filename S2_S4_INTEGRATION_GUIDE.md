# TrustChain S2 + S4 Integration Guide (Task 3)\n\n## Overview\n\nThis document details the integration between S2 (Federated Learning - Amber) and S4 (Frontend/Research - Rishu Kishan) for **Task 3: Integration Sprint (Week 22-26 June)**.\n\nGoal: All components work together. End-to-end flow: User checks in → smart contract records → tokens awarded → FL model receives signal → updated recommendations appear.\n\n## Task 3: S2 + S4 Paired Integration\n\n### Current Status: ✅ COMPLETE\n\n**Deliverables Completed:**\n- ✅ FL model output connected to frontend recommendation feed\n- ✅ Map displays \"Recommended for You\" POIs based on FL scores\n- ✅ 3 different user preference profiles with different recommendations\n- ✅ Frontend switches profiles and shows different recommendations\n- ✅ Check-in system with token tracking\n- ✅ Check-in history visible in UI\n\n### S2 Work - Federated Learning (Amber)\n\n**Files:**\n- `federated/flower_server.py` - FL Server with FedAvg aggregation\n- `federated/flower_client.py` - 3 client nodes for collaborative filtering\n- `federated/task3.py` - Recommendation engine (generates POI scores)\n- `federated/FEDERATED_LEARNING.md` - FL documentation\n\n**Outputs:**\n- `frontend/public/recommendations.json` - Main integration point\n  ```json\n  {\n    \"rounds\": [{\"round\": 1, \"accuracy\": 0.63}, ...],\n    \"profiles\": [\n      {\n        \"id\": \"commuter\",\n        \"label\": \"Transit Commuter\",\n        \"recommendations\": [{\"id\", \"name\", \"score\", ...}, ...]\n      },\n      ...\n    ]\n  }\n  ```\n\n**How It Works:**\n1. 5 rounds of federated averaging improve accuracy from 0.63 to 0.93\n2. Each client maintains category preferences (Transit, Food, Outdoor, etc.)\n3. Global model aggregates across clients\n4. For each profile, recommendations are scored using:\n   - 58% category match (from FL model)\n   - 27% popularity (check-in count)\n   - 15% round boost\n5. Top 8 POIs per profile saved to recommendations.json\n\n### S4 Work - Frontend (Rishu Kishan)\n\n**Files:**\n- `frontend/src/App.jsx` - Main app with state management\n- `frontend/src/PoiMap.jsx` - Leaflet map component\n- `frontend/src/App.css` - Styling\n- `frontend/FRONTEND_GUIDE.md` - Frontend documentation\n\n**Integration Points:**\n\n1. **Load Recommendations** (App.jsx, lines 20-60)\n   ```javascript\n   fetch('/recommendations.json')\n     .then(r => r.json())\n     .then(data => setRecommendationData(data))\n   ```\n\n2. **Display Profile-Specific Recommendations** (App.jsx, lines 175-185)\n   ```javascript\n   const recommendedPois = selectedProfile?.recommendations ??
